## 1. <a name='Introduction'></a>Introduction

We're going to build a toy imperative language with denotational semantics, and show how to verify programs using Hoare logic. We will be using the ideas from the previous to sections meaning to give meaning to programs that may not terminate. With `Part.fix` at our disposal, the meaning of a `while` loop is just a least fixed point, and we can prove things about `while` loops using Scott induction. We will finish with an integer division algorithm written in our toy language, and give a full proof of both termination and correctness using `Part` theory.

## 2. <a name='Language'></a>The language

We will begin with the syntax. The language is the standard textbook IMP, with arithmetic expressions, boolean expressions, and commands.

```lean
inductive aexp where
  | num  : ‚Ąē ‚Üí aexp
  | var  : String ‚Üí aexp
  | nAdd : aexp ‚Üí aexp ‚Üí aexp
  | nSub : aexp ‚Üí aexp ‚Üí aexp
  | nMul : aexp ‚Üí aexp ‚Üí aexp

inductive bexp where
  | bTrue  : bexp
  | bFalse : bexp
  | bAnd   : bexp ‚Üí bexp ‚Üí bexp
  | bOr    : bexp ‚Üí bexp ‚Üí bexp
  | beq    : aexp ‚Üí aexp ‚Üí bexp
  | bLe    : aexp ‚Üí aexp ‚Üí bexp
  | bGt    : aexp ‚Üí aexp ‚Üí bexp
  | bnot   : bexp ‚Üí bexp

inductive Cmd where
  | cskip  : Cmd
  | cseq   : Cmd ‚Üí Cmd ‚Üí Cmd
  | asgn   : String ‚Üí aexp ‚Üí Cmd
  | cif    : bexp ‚Üí Cmd ‚Üí Cmd ‚Üí Cmd
  | cwhile : bexp ‚Üí Cmd ‚Üí Cmd
```

Program state is represented by total functions from variable names to natural numbers, with a pointwise update.

```lean
abbrev State := String ‚Üí ‚Ąē

def State.update (s : State) (x : String) (n : ‚Ąē) : State :=
  fun y => if y = x then n else s y
```

Writing programs as raw `Cmd` constructors is both ugly and tedious, so I added a quotation macro `‚ü™ ... ‚üę` that lets us write programs in a nicely legible syntax, as well as a bunch of other notational conveniences. I won't show them here since it's not interesting but I will be using them moving forward.

```lean
example : Cmd := ‚ü™
  result := 1 ;;
  while x > 0 do
    result := result * x ;;
    x := x - 1
  end
‚üę
```

## 3. <a name='Semantics'></a>Denotational semantics

We've described a grammar for our programs and now we need to state what programs actually compute. The plan is to map each piece of the syntax into a real Lean function. Expressions evaluate to values; commands transform states. Partiality shows up in the semantics of `while`.

Arithmetic and boolean expressions are straightforward.

```lean
def adenote : aexp ‚Üí State ‚Üí ‚Ąē
  | .num n,      _ => n
  | .var x,      s => s x
  | .nAdd a‚āĀ a‚āā, s => adenote a‚āĀ s + adenote a‚āā s
  | .nSub a‚āĀ a‚āā, s => adenote a‚āĀ s - adenote a‚āā s
  | .nMul a‚āĀ a‚āā, s => adenote a‚āĀ s * adenote a‚āā s

def bdenote : bexp ‚Üí State ‚Üí Bool
  | .bTrue,      _ => true
  | .bFalse,     _ => false
  | .bAnd b‚āĀ b‚āā, s => bdenote b‚āĀ s && bdenote b‚āā s
  | .bOr b‚āĀ b‚āā,  s => bdenote b‚āĀ s || bdenote b‚āā s
  | .beq a‚āĀ a‚āā,  s => adenote a‚āĀ s == adenote a‚āā s
  | .bLe a‚āĀ a‚āā,  s => Nat.ble (adenote a‚āĀ s) (adenote a‚āā s)
  | .bGt a‚āĀ a‚āā,  s => Nat.blt (adenote a‚āā s) (adenote a‚āĀ s)
  | .bnot b,     s => !bdenote b s
```

Commands are where partiality enters. A command denotes a partial function `State ‚Üí. State` (ie. `State ‚Üí Part State`). Every case is straightforward except `while`.

```lean
def denote : Cmd ‚Üí State ‚Üí. State
  | .cskip,       s => some s
  | .cseq c‚āĀ c‚āā,  s => do 
    let x ‚Üź denote c‚āĀ s
    denote c‚āā x
  | .asgn x a,    s => some (s.update x (adenote a s))
  | .cif b c‚āĀ c‚āā, s => if bdenote b s then denote c‚āĀ s else denote c‚āā s
  | .cwhile b c,  s =>
      Part.fix (fun (rec : State ‚Üí. State) (s : State) =>
        if bdenote b s then do
          let x ‚Üź denote c s
          rec x
        else some s) s
```

Looking at the `while` case, we're saying that `denote (while b do c)` is the least fixed point of the functional

$$F(\mathrm{rec})(s) = \begin{cases} \mathrm{rec}(‚ü¶c‚üß(s)) & \text{if } ‚ü¶b‚üß(s) \\ s & \text{otherwise.} \end{cases}$$

Remember from part 1, this is the recursive equation a `while` loop satisfies, written as a fixed-point equation. `Part.fix` applies to any functional, so writing the denotation down doesn't require a precondition. Continuity is what guarantees `Part.fix F` actually satisfies $F(\mathrm{Part.fix}\,F) = \mathrm{Part.fix}\,F$, which is what makes this the meaning of a `while` loop and not just some arbitrary partial function. 

Our `F` is ŌČ-Scott continuous since it's built from continuous pieces, and `(State ‚Üí. State)` is a domain, so we have a meaningful least fixed point.

## 4. <a name='Hoare'></a>Hoare logic

We have a programming language, and now we need a way to reason about programs in our language, that is, we need a *program logic*. The canonical choice for sequential imperative programs is [Hoare logic](https://cs.stanford.edu/people/eroberts/courses/soco/projects/2008-09/tony-hoare/logic.html). Hoare logic consists of assertions about the program state and a set of inference rules for reasoning about program state.

Judgments are *triples* $\{P\}\, c\, \{Q\}$ read as "if $P$ holds before running $c$, and $c$ terminates, then $Q$ holds afterward". The rules are as follows.

$$
\frac{}{\{P\}\ \texttt{skip}\ \{P\}}\;\text{(skip)}
\qquad
\frac{}{\{P[a/x]\}\ x := a\ \{P\}}\;\text{(assign)}
$$

$$
\frac{\{P\}\ c_1\ \{Q\} \quad \{Q\}\ c_2\ \{R\}}{\{P\}\ c_1;\,c_2\ \{R\}}\;\text{(seq)}
$$

$$
\frac{\{P \land b\}\ c_1\ \{Q\} \quad \{P \land \neg b\}\ c_2\ \{Q\}}{\{P\}\ \texttt{if } b \texttt{ then } c_1 \texttt{ else } c_2\ \{Q\}}\;\text{(if)}
$$

$$
\frac{\{P \land b\}\ c\ \{P\}}{\{P\}\ \texttt{while } b \texttt{ do } c\ \{P \land \neg b\}}\;\text{(while)}
$$

$$
\frac{P \Rightarrow P' \quad \{P'\}\ c\ \{Q'\} \quad Q' \Rightarrow Q}{\{P\}\ c\ \{Q\}}\;\text{(conseq)}
$$

The `(while)` rule is the loop-invariant rule. To prove $\{P\}\ \texttt{while}\ b\ \texttt{do}\ c\ \{P \land \neg b\}$, find a $P$ that the body preserves whenever the guard holds. Then on exit, $P$ still holds and the guard is false.

Now to encode this in Lean. First, an assertion is a predicate on states.

```lean
abbrev Assertion := State ‚Üí Prop
```

Substitution `P‚ü¶a / x‚üß` is the assertion that `P` holds after we replace `x` with the value of `a`.

```lean
def Assertion.subst (P : Assertion) (x : String) (a : aexp) : Assertion :=
  fun s => P (s.update x (adenote a s))

notation:max P "‚ü¶" a:max " / " x:max "‚üß" => Assertion.subst P x a
```

A Hoare triple `‚¶ÉP‚¶Ą c ‚¶ÉQ‚¶Ą` is partial correctness. Starting from a state satisfying `P`, every state we can reach via `denote c` satisfies `Q`. Diverging executions don't witness anything, so the triple is vacuously satisfied on them.

```lean
def HoareTriple (P Q : Assertion) (c : Cmd) :=
  ‚ąÄ s‚āĀ s‚āā, P s‚āĀ ‚Üí s‚āā ‚ąą (denote c s‚āĀ) ‚Üí Q s‚āā

notation:max "‚¶É" P "‚¶Ą" c:max "‚¶É" Q "‚¶Ą" => HoareTriple P Q c
```

The Hoare derivations themselves are an inductive relation, one constructor per rule. This is exactly the inference system at the top of this section.

```lean
inductive Hoare : Assertion ‚Üí Cmd ‚Üí Assertion ‚Üí Prop where
  | hoare_skip   : ‚ąÄ P,     Hoare P ‚ü™ skip ‚üę P
  | hoare_assign : ‚ąÄ P x a, Hoare (P‚ü¶a / x‚üß) ‚ü™ ‚ÄĻx‚Äļ := ‚ÄĻa‚Äļ ‚üę P
  | hoare_seq : ‚ąÄ P Q R c‚āĀ c‚āā,
      Hoare P c‚āĀ Q ‚Üí
      Hoare Q c‚āā R ‚Üí
      Hoare P ‚ü™ c‚āĀ ;; c‚āā ‚üę R
  | hoare_if : ‚ąÄ P Q b c‚āĀ c‚āā,
      Hoare (P.bexp_true b)  c‚āĀ Q ‚Üí
      Hoare (P.bexp_false b) c‚āā Q ‚Üí
      Hoare P ‚ü™ if b then c‚āĀ else c‚āā end ‚üę Q
  | hoare_while : ‚ąÄ P b c,
      Hoare (P.bexp_true b) c P ‚Üí
      Hoare P ‚ü™ while b do c end ‚üę (P.bexp_false b)
  | hoare_conseq : ‚ąÄ P P' Q Q' c,
      (‚ąÄ s, P s ‚Üí P' s) ‚Üí
      Hoare P' c Q' ‚Üí
      (‚ąÄ s, Q' s ‚Üí Q s) ‚Üí
      Hoare P c Q
```

## 5. <a name='Soundness'></a>Soundness

We now have two ways of talking about programs, using the denotational semantics from ¬ß3 and the Hoare logic from ¬ß4. Soundness tells us that the syntactic Hoare derivations actually mean what they should under the denotational semantics. More precisely,

**Theorem.** `Hoare P c Q ‚Üí ‚¶ÉP‚¶Ą c ‚¶ÉQ‚¶Ą`.

The proof is by induction on the Hoare derivation. Most cases are mechanical unfolding. `simp` together with the definitions of `denote` and `HoareTriple` discharges skip, assignment, sequencing, conditionals, and consequence. The interesting case is `while`. There we have a fixed point and we want to show every state reachable from `denote (while b do c) s‚āĀ` satisfies `P ‚ąß ¬¨b`. 

This is what `fix_induction_mem_of_continuous` from Part 2 was built for, namely Scott induction specialized to membership claims, with the bottom and chain-closure cases discharged for free.

To use it we have to prove the loop functional is ŌČ-Scott continuous. This is mechanical. Each branch of the body is either a constant (`Part.some s`) or a `bind` of a continuous denotation followed by `rec`. `ŌČScottContinuous.bind` and `ŌČScottContinuous.const` from Mathlib's API close both.

```lean
| hoare_while P b c h ih =>
  simp only [HoareTriple, denote]
  intro s‚āĀ s‚āā Ps‚āĀ hfix
  have hF : ŌČScottContinuous
      (fun (rec : State ‚Üí Part State) (s : State) =>
        if bdenote b s then do let x ‚Üź denote c s; rec x else Part.some s) := by
    apply ŌČScottContinuous.of_apply‚āā
    intro s
    by_cases hb : bdenote b s = Bool.true
    ¬∑ simp only [hb, if_true]
      exact ŌČScottContinuous.bind ŌČScottContinuous.const ŌČScottContinuous.id
    ¬∑ simp only [hb, Bool.false_eq_true, if_false]
      exact ŌČScottContinuous.const
  apply @Part.fix_induction_mem_of_continuous _ _ _
    (fun x y => P x ‚Üí P.bexp_false b y) hF _ _ _ hfix Ps‚āĀ
  intro f ih_f x y hy hPx
  split_ifs at hy <;> rename_i hb
  ¬∑ simp [Part.mem_bind_iff] at hy
    obtain ‚ü®z, hz, hyf‚ü© := hy
    exact ih_f z y hyf (ih x z ‚ü®hPx, hb‚ü© hz)
  ¬∑ simp at hy
    subst hy
    exact ‚ü®hPx, hb‚ü©
```

The `apply` reduces the goal to the step case of Scott induction. We split on the loop guard. When the guard is true we ran the body to some intermediate state `z`, and the inductive hypothesis says any further iteration from `z` lands in `P ‚ąß ¬¨b`. When the guard is false we exited with the current state, which already satisfies `P ‚ąß ¬¨b` since we entered with `P` and `b` is now false.

This is the loop-invariant preservation argument you'd write informally on a chalkboard, dropped into the step case of Scott induction with the other two cases handled for us.

## 6. <a name='Division'></a>Worked example: integer division

Since soundness is proven, building a Hoare derivation suffices to prove specific `IMP` programs correct. Integer division by repeated subtraction is the example we will use.

```lean
def div_prog : Cmd := ‚ü™
  q := 0 ;;
  r := m ;;
  while n ‚Č§ r do
    r := r - n ;;
    q := q + 1
  end
‚üę
```

The precondition requires `n > 0` and the postcondition gives the standard division spec.

```lean
def div_pre : Assertion :=
  fun s => s "n" > 0

def div_post : Assertion :=
  fun s => s "m" = s "q" * s "n" + s "r" ‚ąß s "r" < s "n"
```

The loop invariant is the relationship between dividend, divisor, quotient, and remainder, plus the constraint that the divisor is positive (which is needed for the body to behave sensibly).

```lean
def div_inv : Assertion :=
  fun s => s "m" = s "q" * s "n" + s "r" ‚ąß s "n" > 0
```

The derivation follows the control flow of the program. The body is two assignments in sequence, the while rule wraps it, the initialization sits in front, and consequence patches up the pre and post. The little bit of arithmetic comes in when checking the body preserves the invariant. After `simp` we have `r = (q+1) * n + (r - n)` under `n ‚Č§ r`, which `omega` closes.

```lean
theorem div_correct : ‚¶É div_pre ‚¶Ą div_prog ‚¶É div_post ‚¶Ą := by
  apply hoare_sound
  -- ... build the derivation as described above
```

## 7. <a name='Total'></a>Total correctness

Partial correctness is vacuous on diverging programs, which is why the soundness theorem doesn't carry a termination obligation. To prove a program correct we want to show partial correctness *and* termination. `Terminates` says the denotation is defined on every state satisfying the precondition.

```lean
def Terminates (P : Assertion) (c : Cmd) : Prop :=
  ‚ąÄ s, P s ‚Üí (denote c s).Dom

def HoareTotal (P Q : Assertion) (c : Cmd) : Prop :=
  ‚¶ÉP‚¶Ą c ‚¶ÉQ‚¶Ą ‚ąß Terminates P c

notation:max "‚¶É" P "‚¶Ą‚āú " c:max " ‚¶É" Q "‚¶Ą" => HoareTotal P Q c
```

For division, termination reduces to showing `Part.fix divLoopF` is defined on every input satisfying the precondition. Since `divLoopF` is ŌČ-Scott continuous, `Part.fix divLoopF = divLoopF (Part.fix divLoopF)`, so we can unfold the fix one step at a time and recurse on a well-founded measure. The variant for the division loop is `r`, which strictly decreases by at least `n ‚Č• 1` per iteration.

We factor out the loop body as a stand alone `def` and prove it continuous using Mathlib's API. 

```lean
def divStep (t : State) : State :=
  (t.update "r" (t "r" - t "n")).update "q" (t "q" + 1)

def divLoopF : (State ‚Üí Part State) ‚Üí State ‚Üí Part State :=
  fun rec t =>
    if bdenote ‚ü™b| n ‚Č§ r‚üę t then rec (divStep t) else Part.some t

private theorem divLoopF_continuous : ŌČScottContinuous divLoopF := by
  apply ŌČScottContinuous.of_apply‚āā
  intro t
  by_cases hb : bdenote ‚ü™b| n ‚Č§ r‚üę t = Bool.true
  ¬∑ simp only [divLoopF, hb, if_true]
    exact ŌČScottContinuous.apply (divStep t)
  ¬∑ simp only [divLoopF, hb, Bool.false_eq_true, if_false]
    exact ŌČScottContinuous.const
```

The termination lemma is induction on `k` with hypothesis `t "r" ‚Č§ k`. At each step `Part.fix_eq_of_ŌČScottContinuous` rewrites `Part.fix divLoopF t` as `divLoopF (Part.fix divLoopF) t`, and we case on the loop guard. False branch returns `Part.some t`, done. True branch recurses on `divStep t`, whose `r` is strictly smaller.

```lean
private theorem div_loop_dom :
    ‚ąÄ (k : ‚Ąē) (t : State), t "r" ‚Č§ k ‚Üí t "n" > 0 ‚Üí (Part.fix divLoopF t).Dom := by
  intro k
  induction k with
  | zero =>
    intro t hr hn
    rw [Part.fix_eq_of_ŌČScottContinuous divLoopF_continuous]
    have hguard : ¬¨ (bdenote ‚ü™b| n ‚Č§ r‚üę t = Bool.true) := by
      simp [bdenote, adenote]; omega
    simp [divLoopF, hguard]
  | succ k ih =>
    intro t hr hn
    rw [Part.fix_eq_of_ŌČScottContinuous divLoopF_continuous]
    by_cases hguard : bdenote ‚ü™b| n ‚Č§ r‚üę t = Bool.true
    ¬∑ simp only [divLoopF, hguard, if_true]
      have hle : t "n" ‚Č§ t "r" := by simp [bdenote, adenote] at hguard; omega
      have ht'r : (divStep t) "r" ‚Č§ k := by simp [divStep, State.update]; omega
      have ht'n : (divStep t) "n" > 0 := by simp [divStep, State.update]; exact hn
      exact ih (divStep t) ht'r ht'n
    ¬∑ simp [divLoopF, hguard]
```

Then `div_terminates` is mostly definition pushing. Identify the denotation of `div_prog` with `Part.fix divLoopF` applied to the post-init state, then invoke `div_loop_dom`. Total correctness follows from combining `div_correct` and `div_terminates`.

```lean
theorem div_terminates : Terminates div_pre div_prog := by
  intro s hn_pre
  let s' : State := (s.update "q" 0).update "r" (s "m")
  have hs'n : s' "n" > 0 := by simp [s', State.update]; exact hn_pre
  have prog_eq : denote div_prog s = Part.fix divLoopF s' := by
    unfold divLoopF
    simp [div_prog, denote, State.update, adenote, s', divStep]
    rfl
  rw [prog_eq]
  exact div_loop_dom (s' "r") s' (le_refl _) hs'n

theorem div_total : ‚¶É div_pre ‚¶Ą‚āú div_prog ‚¶É div_post ‚¶Ą :=
  ‚ü®div_correct, div_terminates‚ü©
```
